## 1. What Is CRT?

Chinese Remainder Theorem helps solve systems like:

```text
x ≡ a1 (mod m1)
x ≡ a2 (mod m2)
x ≡ a3 (mod m3)
```

It finds a number `x` that satisfies all equations.

Example:

```text
x ≡ 2 (mod 3)
x ≡ 3 (mod 5)
x ≡ 2 (mod 7)
```

One solution is:

```text
x = 23
```

Check:

```text
23 % 3 = 2
23 % 5 = 3
23 % 7 = 2
```

---

## 2. Why This Topic Matters

CRT appears in:

- number theory
- modular equations
- competitive programming
- cryptography
- calendar cycles
- clock problems
- distributed systems hashing
- reconstruction from residues
- optimization under multiple periodic constraints

In CP, CRT is usually used when the problem says:

```text
Find x such that x leaves different remainders under different moduli.
```

---

## 3. Core Problem Form

Given:

```text
x ≡ r1 mod m1
x ≡ r2 mod m2
...
x ≡ rk mod mk
```

Find:

```text
x
```

Usually smallest non-negative solution.

---

## 4. Simple Intuition

Each congruence says:

```text
x belongs to a repeating arithmetic sequence
```

Example:

```text
x ≡ 2 mod 3
```

means:

```text
2, 5, 8, 11, 14, 17, 20, 23, ...
```

Another:

```text
x ≡ 3 mod 5
```

means:

```text
3, 8, 13, 18, 23, ...
```

Common number:

```text
23
```

CRT gives a direct way to find it without brute force.

---

## 5. When CRT Works

### Case 1 — Pairwise coprime moduli

If:

```text
gcd(mi, mj) = 1 for every i != j
```

Then:

```text
solution always exists
```

and is unique modulo:

```text
M = m1 * m2 * ... * mk
```

### Case 2 — Non-coprime moduli

Solution exists only if every pair is compatible:

```text
r1 ≡ r2 mod gcd(m1, m2)
```

If not compatible:

```text
no solution
```

---

## 6. Pairwise Coprime Moduli

For most CP beginner/intermediate CRT problems, moduli are pairwise coprime.

Example:

```text
3, 5, 7
```

Because:

```text
gcd(3,5)=1
gcd(3,7)=1
gcd(5,7)=1
```

Product:

```text
M = 3 * 5 * 7 = 105
```

Answer will be unique modulo 105.

So if one solution is:

```text
x = 23
```

then all solutions are:

```text
23 + 105k
```

---

## 7. Core Formula

For pairwise coprime moduli:

```text
M = m1 * m2 * ... * mk
```

For each equation:

```text
Mi = M / mi
```

Find inverse:

```text
inv_i = inverse(Mi mod mi)
```

Then:

```text
x = Σ ai * Mi * inv_i  mod M
```

---

## 8. Step-by-Step CRT Construction

Problem:

```text
x ≡ 2 mod 3
x ≡ 3 mod 5
x ≡ 2 mod 7
```

Step 1:

```text
M = 3 * 5 * 7 = 105
```

Step 2:

For modulus 3:

```text
M1 = 105 / 3 = 35
Need inverse of 35 mod 3
35 % 3 = 2
inverse of 2 mod 3 = 2
term = 2 * 35 * 2 = 140
```

Step 3:

For modulus 5:

```text
M2 = 105 / 5 = 21
21 % 5 = 1
inverse of 1 mod 5 = 1
term = 3 * 21 * 1 = 63
```

Step 4:

For modulus 7:

```text
M3 = 105 / 7 = 15
15 % 7 = 1
inverse of 1 mod 7 = 1
term = 2 * 15 * 1 = 30
```

Step 5:

```text
sum = 140 + 63 + 30 = 233
```

Step 6:

```text
x = 233 mod 105 = 23
```

Answer:

```text
23
```

## 9. Extended Euclid Connection

CRT needs modular inverse.

For:

```text
inverse(a mod m)
```

We need:

```text
a * x ≡ 1 mod m
```

This exists only when:

```text
gcd(a, m) = 1
```

Extended Euclid finds `x`.

## 10. Problem Form 1 — Two Congruences

### Problem

Solve:

```text
x ≡ 2 mod 3
x ≡ 3 mod 5
```

---

### Step-by-Step Working

Sequence from first:

```text
2, 5, 8, 11, 14, 17, 20, 23, ...
```

Check modulo 5:

```text
2 % 5 = 2
5 % 5 = 0
8 % 5 = 3
```

So:

```text
x = 8
```

Unique modulo:

```text
3 * 5 = 15
```

All solutions:

```text
8 + 15k
```

---

### C++ Code

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {

    int r1 = 2, m1 = 3;
    int r2 = 3, m2 = 5;

    for (int x = 0; x < m1 * m2; x++) {
        if (x % m1 == r1 && x % m2 == r2) {
            cout << x << endl;
            break;
        }
    }

    return 0;
}
```

## 11. Problem Form 2 — Multiple Congruences

### Problem

Solve:

```text
x ≡ 2 mod 3
x ≡ 3 mod 5
x ≡ 2 mod 7
```

---

### Pattern

```text
CRT with pairwise coprime moduli
```

---

### Answer

```text
23
```

---

### C++ Code

```cpp
#include <bits/stdc++.h>
using namespace std;

long long extendedGcd(long long a, long long b, long long &x, long long &y) {

    if (b == 0) {
        x = 1;
        y = 0;
        return a;
    }

    long long x1, y1;

    long long g = extendedGcd(b, a % b, x1, y1);

    x = y1;
    y = x1 - (a / b) * y1;

    return g;
}

long long modInverse(long long a, long long mod) {

    long long x, y;

    long long g = extendedGcd(a, mod, x, y);

    if (g != 1) return -1;

    x %= mod;
    if (x < 0) x += mod;

    return x;
}

int main() {

    vector<long long> rem = {2, 3, 2};
    vector<long long> mod = {3, 5, 7};

    long long M = 1;

    for (long long m : mod) {
        M *= m;
    }

    long long ans = 0;

    for (int i = 0; i < rem.size(); i++) {

        long long Mi = M / mod[i];

        long long inv = modInverse(Mi % mod[i], mod[i]);

        ans = (ans + rem[i] * Mi * inv) % M;
    }

    cout << ans << endl; // 23

    return 0;
}
```

## 12. Problem Form 3 — Calendar / Cycle Problems

### Problem

An event happens every 3 days with offset 2.

Another event happens every 5 days with offset 3.

Find the first day both conditions match.

---

### Mapping

```text
day ≡ 2 mod 3
day ≡ 3 mod 5
```

This is CRT.

---

### Step-by-Step Working

From first cycle:

```text
2, 5, 8, 11, 14, ...
```

Check second:

```text
8 % 5 = 3
```

Answer:

```text
8
```

Next repeats after:

```text
LCM(3,5)=15
```

So all days:

```text
8, 23, 38, 53, ...
```

```text
remainder 1 when divided by 2
```

---

### Equation

```text

x ≡ 2 mod 3
```

### Step-by-Step Working

Try solution using CRT.

Product:

```text
M = 2 * 3 * 5 = 30
```

Answer is unique modulo 30.

Testing values:

```text
23 % 2 = 1
23 % 3 = 2
23 % 5 = 3
```

Answer:

```text
23
```

## 14. Problem Form 5 — Non-Coprime Moduli

### Problem

Solve:

```text
x ≡ 2 mod 6
x ≡ 8 mod 10
```

Moduli are not coprime:

```text
gcd(6,10)=2
```

Check compatibility:

```text
2 % 2 = 0

```

Compatible.

So solution exists.

### Impossible Example

```text

```

Check:

```text
gcd(6,10)=2
2 % 2 = 0
3 % 2 = 1
```

Not compatible.

So:

```text
no solution
```

Solve:

```text
x ≡ 1 mod 2
x ≡ 2 mod 3

```

```text
M = 2 * 3 * 5 = 30
```

Step 2:

For modulus 2:

```text
M1 = 30 / 2 = 15

inverse = 1
term = 1 * 15 * 1 = 15
```

Step 3:

For modulus 3:

```text
M2 = 30 / 3 = 10
10 % 3 = 1
inverse = 1
term = 2 * 10 * 1 = 20
```

Step 4:

For modulus 5:

```text
M3 = 30 / 5 = 6
6 % 5 = 1
inverse = 1
term = 3 * 6 * 1 = 18
```

Step 5:

```text
sum = 15 + 20 + 18 = 53
```

Step 6:

```text
53 mod 30 = 23
```

```text
23
```

## 16. Common Mistakes

### Mistake 1 — Forgetting coprime condition

Basic CRT formula assumes:

```text
pairwise coprime moduli
```

If not, use generalized CRT.

---

### Mistake 2 — Using normal division under modulo

Wrong:

```text
x / y mod m
```

Correct:

```text
x * inverse(y) mod m
```

---

### Mistake 3 — Not normalizing negative answer

Always do:

```cpp
ans = (ans % M + M) % M;
```

---

### Mistake 4 — Overflow in product

If moduli are large:

```text
M = m1 * m2 * ...
```

can overflow `long long`.

Use `__int128` if needed.

---

### Mistake 5 — Confusing remainder and modulus

Equation format:

```text
x ≡ remainder mod modulus
```

Not the other way around.

---

## 17. Decision Tree

```text
Modulo equations?
|
+-- Single equation?
|   |
|   +-- Use modular arithmetic directly
|
+-- Multiple equations?
    |
    +-- Pairwise coprime moduli?
    |   |
    |   +-- Yes
    |       |
    |       +-- Use basic CRT
    |
    +-- Not pairwise coprime?
        |
        +-- Check compatibility:
        |   remainder difference divisible by gcd
        |
        +-- Compatible?
            |
            +-- Yes -> Generalized CRT
            |
            +-- No -> No solution
```

## 19. Complexity

For `k` equations:

```text
O(k log M)
```

because each inverse uses Extended Euclid.

Brute force:

```text
O(product of moduli)
```

Avoid brute force unless values are tiny.

## 20. Reusable C++ Template

```cpp
#include <bits/stdc++.h>
using namespace std;

long long extendedGcd(long long a, long long b, long long &x, long long &y) {

    if (b == 0) {
        x = 1;
        y = 0;
        return a;
    }

    long long x1, y1;

    long long g = extendedGcd(b, a % b, x1, y1);

    x = y1;
    y = x1 - (a / b) * y1;

    return g;
}

long long modInverse(long long a, long long mod) {

    long long x, y;

    long long g = extendedGcd(a, mod, x, y);

    if (g != 1) {
        return -1;
    }

    x %= mod;

    if (x < 0) x += mod;

    return x;
}

// Works when moduli are pairwise coprime.
long long crtCoprime(vector<long long> rem, vector<long long> mod) {

    long long M = 1;

    for (long long m : mod) {
        M *= m;
    }

    long long ans = 0;

    for (int i = 0; i < rem.size(); i++) {

        long long Mi = M / mod[i];

        long long inv = modInverse(Mi % mod[i], mod[i]);

        ans = (ans + rem[i] * Mi % M * inv % M) % M;
    }

    ans = (ans % M + M) % M;

    return ans;
}

int main() {

    vector<long long> rem = {2, 3, 2};
    vector<long long> mod = {3, 5, 7};

    cout << crtCoprime(rem, mod) << endl; // 23

    return 0;
}
```

## 21. CP / FAANG Problem Forms

---

### Problem 1 — Solve Multiple Remainders

#### Problem

Find the smallest non-negative `x`:

```text
x ≡ 2 mod 3
x ≡ 3 mod 5
x ≡ 2 mod 7
```

---

#### Pattern

```text
Basic CRT
```

---

#### Step-by-Step Working

Step 1:

```text
M = 3 * 5 * 7 = 105
```

Step 2:

```text
M1 = 105/3 = 35
M2 = 105/5 = 21
M3 = 105/7 = 15
```

Step 3:

```text
inverse(35 mod 3) = inverse(2 mod 3) = 2
inverse(21 mod 5) = inverse(1 mod 5) = 1
inverse(15 mod 7) = inverse(1 mod 7) = 1
```

Step 4:

```text
x = 2*35*2 + 3*21*1 + 2*15*1
  = 233
```

Step 5:

```text
233 % 105 = 23
```

Answer:

```text
23
```

---

#### Commented C++ Code

```cpp
#include <bits/stdc++.h>
using namespace std;

long long extendedGcd(long long a, long long b, long long &x, long long &y) {

    if (b == 0) {
        x = 1;
        y = 0;
        return a;
    }

    long long x1, y1;

    long long g = extendedGcd(b, a % b, x1, y1);

    x = y1;
    y = x1 - (a / b) * y1;

    return g;
}

long long modInverse(long long a, long long mod) {

    long long x, y;

    long long g = extendedGcd(a, mod, x, y);

    if (g != 1) return -1;

    x %= mod;
    if (x < 0) x += mod;

    return x;
}

int main() {

    vector<long long> rem = {2, 3, 2};
    vector<long long> mod = {3, 5, 7};

    long long M = 1;

    for (long long m : mod) M *= m;

    long long ans = 0;

    for (int i = 0; i < rem.size(); i++) {

        // product of all moduli except current one
        long long Mi = M / mod[i];

        // inverse makes Mi become 1 under current modulus
        long long inv = modInverse(Mi % mod[i], mod[i]);

        // add contribution of current equation
        ans = (ans + rem[i] * Mi % M * inv % M) % M;
    }

    ans = (ans + M) % M;

    cout << ans << endl;

    return 0;
}
```

### Problem 2 — First Day Matching Multiple Cycles

#### Problem

A process satisfies:

```text
day ≡ 1 mod 2
day ≡ 2 mod 3
day ≡ 3 mod 5
```

Find the first valid day.

---

#### Pattern

```text
CRT cycle alignment
```

---

#### Step-by-Step Working

Step 1:

```text
M = 2 * 3 * 5 = 30
```

Step 2:

Use CRT.

Answer:

```text
23
```

Step 3:

All future days:

```text
23 + 30k
```

---

#### Commented C++ Code

```cpp
#include <bits/stdc++.h>
using namespace std;

long long egcd(long long a, long long b, long long &x, long long &y) {

    if (b == 0) {
        x = 1;
        y = 0;
        return a;
    }

    long long x1, y1;

    long long g = egcd(b, a % b, x1, y1);

    x = y1;
    y = x1 - (a / b) * y1;

    return g;
}

long long inv(long long a, long long m) {

    long long x, y;

    long long g = egcd(a, m, x, y);

    if (g != 1) return -1;

    return (x % m + m) % m;
}

int main() {

    vector<long long> r = {1, 2, 3};
    vector<long long> m = {2, 3, 5};

    long long M = 1;

    for (auto x : m) M *= x;

    long long ans = 0;

    for (int i = 0; i < r.size(); i++) {

        long long part = M / m[i];

        long long inverse = inv(part % m[i], m[i]);

        ans = (ans + r[i] * part % M * inverse % M) % M;
    }

    cout << ans << endl;

    return 0;
}
```

### Problem 3 — Check If Non-Coprime CRT Has Solution

#### Problem

Check whether this system has a solution:

```text
x ≡ 2 mod 6
x ≡ 3 mod 10
```

---

#### Pattern

```text
Compatibility check using gcd
```

---

#### Step-by-Step Working

Step 1:

```text
g = gcd(6,10) = 2
```

Step 2:

Check remainder difference:

```text
3 - 2 = 1
```

Step 3:

```text
1 % 2 != 0
```

So:

```text
no solution
```

---

#### Commented C++ Code

```cpp
#include <bits/stdc++.h>
using namespace std;

bool compatible(long long r1, long long m1, long long r2, long long m2) {

    long long g = gcd(m1, m2);

    // Compatible only if both remainders agree modulo gcd.
    return ((r2 - r1) % g + g) % g == 0;
}

int main() {

    long long r1 = 2, m1 = 6;
    long long r2 = 3, m2 = 10;

    if (compatible(r1, m1, r2, m2)) {
        cout << "Solution exists\n";
    } else {
        cout << "No solution\n";
    }

    return 0;
}
```

### Problem 4 — Brute Force Small CRT

#### Problem

For small values, find `x` by checking all values.

---

#### Pattern

```text
Brute force CRT
```

---

#### Step-by-Step Working

For:

```text
x ≡ 2 mod 3
x ≡ 3 mod 5
```

Try:

```text
0,1,2,3,4,5,6,7,8
```

At:

```text
x = 8
```

both match.

---

#### Commented C++ Code

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {

    vector<int> rem = {2, 3};
    vector<int> mod = {3, 5};

    int product = 1;

    for (int m : mod) product *= m;

    for (int x = 0; x < product; x++) {

        bool ok = true;

        for (int i = 0; i < rem.size(); i++) {

            if (x % mod[i] != rem[i]) {
                ok = false;
                break;
            }
        }

        if (ok) {
            cout << x << endl;
            break;
        }
    }

    return 0;
}
```

### Problem 5 — Merge Two Congruences Incrementally

#### Problem

Merge:

```text
x ≡ r1 mod m1
x ≡ r2 mod m2
```

into one congruence:

```text
x ≡ R mod L
```

---

#### Pattern

```text
General CRT merge
```

---

#### Step-by-Step Working

The merged modulus is:

```text
lcm(m1,m2)
```

Only possible if:

```text
(r2-r1) % gcd(m1,m2) == 0
```

---

#### Commented C++ Code

```cpp
#include <bits/stdc++.h>
using namespace std;

long long egcd(long long a, long long b, long long &x, long long &y) {

    if (b == 0) {
        x = 1;
        y = 0;
        return a;
    }

    long long x1, y1;

    long long g = egcd(b, a % b, x1, y1);

    x = y1;
    y = x1 - (a / b) * y1;

    return g;
}

// Merge two congruences:
// x ≡ r1 mod m1
// x ≡ r2 mod m2
bool mergeCRT(long long r1, long long m1,
              long long r2, long long m2,
              long long &R, long long &M) {

    long long x, y;

    long long g = egcd(m1, m2, x, y);

    long long diff = r2 - r1;

    if (diff % g != 0) return false;

    long long lcm = m1 / g * m2;

    long long t = (diff / g * x) % (m2 / g);

    R = (r1 + m1 * t) % lcm;

    if (R < 0) R += lcm;

    M = lcm;

    return true;
}

int main() {

    long long R, M;

    bool ok = mergeCRT(2, 6, 8, 10, R, M);

    if (!ok) {
        cout << "No solution\n";
    } else {
        cout << "x ≡ " << R << " mod " << M << "\n";
    }

    return 0;
}
```
